# Uptime Logpush

[English](README.md) | [简体中文](README_zh.md)

## 介绍
Uptime Logpush 是一款移动端监控工具，专为对接 Uptime Kuma 设计。它能够通过 Webhook 拉取网页服务器上生成的日志文件，并在本地设备上提供实时的状态更新与系统通知。该应用让您无需持续打开网页端即可监控服务器节点的运行状况。

## 功能特性
- **实时日志流式获取**：主动监听并从配置的数据源拉取日志数据。
- **状态解析与展示**：自动解析日志条目，展示服务器状态（UP/DOWN）、响应时间、主机名、IP 地址以及监听的端口号。
- **灵活的数据源配置**：支持从标准的纯文本日志 URL 或直接的 Uptime Kuma Webhook 端点获取数据。
- **后台实时监控**：利用前台服务保持应用活跃，即便应用在后台运行，数据同步依然能够持续进行。
- **可自定义轮询间隔**：允许用户手动设置后台拉取接口的更新频率（如每 300 秒）。高频设置能提供更好的实时性，但会略微增加设备耗电。
- **系统通知推送**：一旦检测到新的日志更新，会在设备通知栏中触发推送通知提醒。
- **隐私保护模式**：支持将该应用从设备的最近任务列表中隐藏，以增强个人隐私和安全性。

## 界面导航
应用底部导航栏提供了三个主要功能选项卡：
- **日志 (Logs)**：主要的监控界面。实时展示服务器状态更新流，包括 UP/DOWN 状态、响应时间以及具体连接信息（主机名、端口）。
- **历史 (History)**：展示历史日志记录，用于回顾过去服务器状态的变化情况。
- **配置 (Config)**：用于访问和修改所有系统设置。

## 配置详情
在应用的 **配置 (Config)** 选项卡中，您可以找到以下设置项：

- **日志拉取源 URL**：在此输入您的日志获取 URL 或 Webhook 地址。该字段同时支持标准的纯文本格式和 Uptime Kuma 的 Webhook 数据结构。
- **后台实时监控**：开启此开关，应用将启动前台服务。这能确保应用退至后台时，依然能够实时拉取数据并发送通知。
- **实时拉取轮询间隔**：设置每次数据请求之间的时间间隔（单位：秒）。您可以从预设列表中选择，也可以输入自定义数值。
- **有新日志时推送系统通知**：开启此开关，当监控的服务器产生新的日志更新时，系统会在设备通知栏中显示通知提醒。
- **在最近任务列表中隐藏**：开启此开关，应用将不会出现在系统的最近应用列表中，进一步加强隐私保护。

## 使用指南
1. 在您的移动设备上下载并安装 Uptime Logpush。
2. 打开应用，进入底部的 **配置 (Config)** 选项卡。
3. 在 **日志拉取源 URL** 中输入您的 Uptime Kuma 日志文件地址。
4. 设置您偏好的 **实时拉取轮询间隔**（例如 300 秒）。
5. 确保 **后台实时监控** 开关已打开，以保持服务持续运行。
6. 完成后，您应该就能在 **日志 (Logs)** 选项卡中看到服务器的实时状态了。
